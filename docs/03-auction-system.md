# 경매 자동 분석 시스템 - 부동산 경매 통합 플랫폼

## 한줄 요약
온비드/법원경매/신탁사/파산경매 4개 소스를 통합 수집하고, 시세 교차검증 + 리스크 분석 + React 대시보드로 시각화하는 경매 분석 시스템

## 프로젝트 구성 (5개 연관 프로젝트)

### 1. auction-sniper-v2 (메인 통합기)
**역할**: 멀티소스 경매물건 수집 → 필터링 → 시세 검증 → HTML 리스팅

**데이터 소스 & 규모**:
| 소스 | 수집량 | 방식 |
|------|--------|------|
| 온비드 모바일 API | 99건 | REST API |
| 파산경매 (scourt) | 77건 | PDF 다운로드 + 파싱 |
| 신탁사 경매 (5개사) | 130건 | 웹 크롤링 |
| **합계** | **306건** | |

**시세 교차검증**:
- Disco API: 공시가격 + 실거래가
- data.go.kr: 시군구 실거래 데이터
- 온비드 OpenAPI: 권리 정보 (근저당, 압류, 보증금)

**비용 모델 자동 계산**:
- 취득세 4.6%, 등록비 ~200만, 중개수수료 0.4-0.5%, 법무비 ~50만

### 2. auction_automation (법원경매 + React 대시보드)
**역할**: 경매마당(madangs.com) API 기반 수집 + Disco 시세 분석 + React 대시보드

**파이프라인**:
```
마당 로그인 → 검색 API (지역/용도 필터)
→ 상세 API (8스레드 병렬) + 좌표 추출
→ 1차 필터: 최저가 < 감정가 × 130%
→ [선택] Disco 시세 보강
→ React 대시보드 시각화
```

**대시보드 기능**:
- 물건 카드 (이미지 + 감정가 + 낙찰률)
- 다중 필터: 지역, 용도, 가격, 비율, 유찰횟수
- 거래 탭: 반경 300m/500m/1km 인근 실거래 + 5분위 통계
- Disco 분석 탭: 시세 이력, 토지/건물 분리
- Google Maps 위성뷰

**성능**: Playwright → HTTP API 전환으로 10배 속도 향상

### 3. bankrupty_autcion (파산경매 AI 분석)
**역할**: 법원 파산경매 공고 PDF → Claude AI 구조화 → 대시보드

**프로세스**:
```
법원 웹사이트 PDF 다운로드 (90건)
→ PyPDF2 텍스트 추출 (처음 3페이지)
→ Claude Sonnet AI 분석 (8.5초/건)
→ JSON 구조화 (87건)
→ HTML 대시보드 (카테고리/가격 필터)
```

**최적화 결과**:
- TXT 추출 vs 직접 PDF: 토큰 78% 절감 ($0.80 vs $3.60)
- 순차 vs 병렬: 순차가 41% 더 빠름 (8.5s vs 14.6s)
- Sonnet vs Haiku: Haiku 품질 60% 하락 → Sonnet 유지

**데이터 분포** (87건):
- 채권 35.6%, 부동산 27.6%, 기타 21.8%, 유체동산 8%, 상표권 5.7%

### 4. auction-sniper v1 (프로토타입)
**역할**: 초기 프레임워크 — 자동차, 중고품 등 멀티마켓 지원
- Naver 부동산, 엔카, 번개장터, 당근마켓 시세 조회
- SQLite DB, Rich CLI

### 5. auction_crawler (레거시 크롤러)
**역할**: 실험용 Selenium 크롤러 + OCR 추출
- 인천 빌라 크롤링 (Selenium)
- 코레일 예약 자동화 (부수 프로젝트)

## 통합 기술 스택
| 영역 | 기술 |
|------|------|
| 수집 | Python (urllib, concurrent.futures, Playwright) |
| 시세 분석 | Disco API, Naver 부동산, data.go.kr |
| AI 분석 | Claude Sonnet (PDF 구조화) |
| 프론트엔드 | React 19 + Vite + Tailwind + Google Maps |
| 데이터 | JSON (파일 기반), SQLite (v1) |
| PDF 처리 | PyPDF2, pdfplumber |
